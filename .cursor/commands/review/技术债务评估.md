# 技术债务评估

请对当前项目进行技术债务评估，识别和量化各类技术债务：

## 1. 代码债务识别
- 扫描重复代码块和相似逻辑
- 识别过度复杂的方法和类（圈复杂度>10）
- 检查命名不规范和代码异味
- 分析过长的方法和参数过多的函数

## 2. 架构债务分析
- 识别模块间的强耦合和循环依赖
- 检查违反SOLID原则的设计
- 分析不合理的架构决策和技术选型
- 评估系统的可扩展性和维护性问题

## 3. 测试债务评估
- 检查测试覆盖率不足的模块
- 识别缺失的单元测试和集成测试
- 分析测试代码的质量和维护成本
- 评估测试自动化程度

## 4. 文档债务检查
- 检查缺失或过时的代码注释
- 识别缺少的API文档和设计文档
- 分析README和部署文档的完整性
- 评估知识传承和新人上手难度

## 5. 依赖债务审计
- 扫描过时的第三方依赖包
- 识别存在安全漏洞的依赖
- 检查未使用的依赖和冗余包
- 评估依赖升级的风险和成本

## 6. 债务量化和优先级
- 估算每类债务的修复工作量（人天）
- 评估债务对业务的影响程度（高/中/低）
- 计算债务的修复收益和ROI
- 制定债务清理的优先级和时间计划

请提供具体的债务清单、量化评估和可执行的清理方案。

## 输出要求：
**请将评估结果保存到 `.cursor/技术债务报告.md` 文件中，包含：**
- 技术债务分类和量化评估
- 债务清理优先级和时间计划
- 具体的修复方案和代码示例
- 债务预防措施和最佳实践

这样AI助手就能持续跟踪项目的技术债务状况。
