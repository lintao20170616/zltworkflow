# 依赖管理分析

分析项目依赖的安全性和合规性：

## 1. 依赖清单分析
- 直接和间接依赖统计
- 依赖版本新旧程度
- 过时依赖识别

## 2. 安全性评估
- CVE安全漏洞扫描
- 依赖包维护状态
- 供应链攻击风险

## 3. 使用合理性
- 未使用依赖（僵尸依赖）
- 功能重复的依赖
- 包大小影响评估

## 4. 许可证合规
- 开源许可证类型
- 许可证冲突风险
- 商业使用限制

## 5. 依赖关系分析
- 依赖树复杂度
- 版本冲突识别
- 传递依赖影响

## 6. 改进建议
- 依赖清理方案
- 安全漏洞修复
- 更新策略建议

## 输出要求：
**保存到 `.cursor/依赖管理报告.md`，包含：**
- 依赖风险评估
- 升级和清理建议
- 依赖管理最佳实践
