# 业务逻辑提取

请从当前项目代码中提取和梳理核心业务逻辑：

## 1. 核心业务流程识别
- 分析Controller和Service层的主要业务方法
- 识别关键的业务用例和操作流程
- 梳理用户交互的主要路径
- 提取工作流和状态转换逻辑

## 2. 业务规则挖掘
- 从条件判断中提取业务规则（if/else, switch）
- 识别数据验证和业务约束条件
- 提取计算规则和公式逻辑
- 分析权限控制和访问规则

## 3. 数据模型分析
- 识别核心业务实体和关系
- 分析数据模型的约束和验证规则
- 梳理实体间的关联关系
- 提取数据生命周期管理规则

## 4. 业务决策点识别
- 识别关键的业务决策逻辑
- 分析决策的输入条件和输出结果
- 梳理异常处理和边界情况
- 提取业务配置和参数设置

## 5. 集成和接口逻辑
- 分析第三方系统集成的业务逻辑
- 提取API接口的业务含义
- 识别数据同步和转换规则
- 梳理消息队列和事件处理逻辑

## 6. 业务文档生成
- 生成业务流程图和时序图
- 编写业务规则说明文档
- 创建数据字典和实体关系图
- 整理业务术语和概念定义

## 输出格式要求：
- 使用结构化的文档格式
- 包含具体的代码位置引用
- 提供业务规则的自然语言描述
- 标注业务逻辑的重要程度和复杂度

请基于实际代码内容进行分析，重点关注业务价值高的核心逻辑。

## 输出要求：
**请将业务分析结果保存到 `.cursor/业务逻辑文档.md` 文件中，包含：**
- 核心业务流程图和时序图
- 业务规则和约束条件清单
- 数据模型和实体关系图
- 业务术语词典和概念定义

这样AI助手就能快速理解项目的业务背景和核心逻辑。
