# 架构分析

分析项目的系统架构设计：

## 1. 架构风格识别
- 架构模式（单体/微服务/分层）
- 组件划分和职责
- 架构决策评估

## 2. 分层设计分析
- 层次结构和依赖关系
- 模块边界定义
- 耦合度评估

## 3. 数据架构
- 数据存储设计
- 数据流和访问模式
- 一致性和事务处理

## 4. 集成和接口
- 系统集成方式
- API设计质量
- 外部依赖分析

## 5. 架构问题和改进
- 架构风险点
- 改进建议
- 重构优先级

## 输出要求：
**保存到 `.cursor/系统架构文档.md`，包含：**
- 架构概览和组件关系
- 主要架构问题
- 改进建议和实施方案
