# 示例：使用文件进行规划的实际应用

## 示例 1：研究任务

**用户请求：** "研究晨练的好处并写一份总结"

### 循环 1：创建计划
```bash
写入 task_plan.md
```

```markdown
# 任务计划：晨练好处研究

## 目标
创建一份关于晨练好处的研究总结。

## 阶段
- [ ] 阶段 1: 创建此计划 ✓
- [ ] 阶段 2: 搜索和收集来源
- [ ] 阶段 3: 综合发现
- [ ] 阶段 4: 交付总结

## 关键问题
1. 有哪些身体健康好处？
2. 有哪些心理健康好处？
3. 有哪些科学研究支持这一点？

## 状态
**当前处于阶段 1** - 创建计划中
```

### 循环 2：研究
```bash
阅读 task_plan.md           # 刷新目标
网络搜索 "晨练的好处"
写入 notes.md              # 存储发现
编辑 task_plan.md           # 标记阶段 2 完成
```

### 循环 3：综合
```bash
阅读 task_plan.md           # 刷新目标
阅读 notes.md               # 获取发现
写入 morning_exercise_summary.md
编辑 task_plan.md           # 标记阶段 3 完成
```

### 循环 4：交付
```bash
阅读 task_plan.md           # 验证完成
交付 morning_exercise_summary.md
```

## 示例 4：错误恢复模式

当某些操作失败时，不要隐藏它：

### 之前（错误）
```
操作：读取 config.json
错误：文件未找到
操作：读取 config.json  # 静默重试
操作：读取 config.json  # 再次重试
```

### 之后（正确）
```
操作：读取 config.json
错误：文件未找到

# 更新 task_plan.md:
## 遇到的错误
- config.json 未找到 → 将创建默认配置

操作：写入 config.json（默认配置）
操作：读取 config.json
成功！
```

## 决定前先阅读模式

**在重大决策前始终阅读你的计划：**

```
[已经发生了许多工具调用...]
[上下文越来越长...]
[原始目标可能被遗忘...]

→ 阅读 task_plan.md          # 这会将目标重新带回注意力！
→ 现在做决定                 # 目标在上下文中是新鲜的
```

这就是为什么 Manus 可以处理约 50 个工具调用而不会迷失方向。计划文件充当"目标刷新"机制。
