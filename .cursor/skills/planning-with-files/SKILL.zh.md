# 使用文件进行规划

像 Manus 一样工作：使用持久化的 markdown 文件作为你的"磁盘上的工作记忆"。

## 快速开始

在任何复杂任务之前：

1. **在工作目录中创建 `task_plan.md`**
2. **定义阶段**，使用复选框
3. **每个阶段后更新** - 标记 [x] 并更改状态
4. **决定前先阅读** - 在注意力窗口中刷新目标

## 三文件模式

对于每个非平凡任务，创建三个文件：

| 文件 | 用途 | 何时更新 |
|------|------|----------|
| `task_plan.md` | 跟踪阶段和进度 | 每个阶段后 |
| `notes.md` | 存储发现和研究 | 研究期间 |
| `[deliverable].md` | 最终输出 | 完成时 |

## 核心工作流程

```
循环 1: 创建 task_plan.md，包含目标和阶段
循环 2: 研究 → 保存到 notes.md → 更新 task_plan.md
循环 3: 阅读 notes.md → 创建交付物 → 更新 task_plan.md
循环 4: 交付最终输出
```

### 循环详解

**每次主要操作前：**
```bash
阅读 task_plan.md  # 在注意力窗口中刷新目标
```

**每个阶段后：**
```bash
编辑 task_plan.md  # 标记 [x]，更新状态
```

**存储信息时：**
```bash
写入 notes.md     # 不要塞满上下文，存储在文件中
```

## task_plan.md 模板

对于任何复杂任务，首先创建此文件：

```markdown
# 任务计划：[简要描述]

## 目标
[一句话描述最终状态]

## 阶段
- [ ] 阶段 1: 规划和设置
- [ ] 阶段 2: 研究/收集信息
- [ ] 阶段 3: 执行/构建
- [ ] 阶段 4: 审查和交付

## 关键问题
1. [要回答的问题]
2. [要回答的问题]

## 已做决策
- [决策]: [理由]

## 遇到的错误
- [错误]: [解决方案]

## 状态
**当前处于阶段 X** - [我现在在做什么]
```

## notes.md 模板

用于研究和发现：

```markdown
# 笔记：[主题]

## 来源

### 来源 1: [名称]
- URL: [链接]
- 关键点:
  - [发现]
  - [发现]

## 综合发现

### [类别]
- [发现]
- [发现]
```

## 关键规则

### 1. 始终先创建计划
永远不要在复杂任务开始时没有 `task_plan.md`。这是不可协商的。

### 2. 决定前先阅读
在任何重大决策之前，阅读计划文件。这使目标保持在你的注意力窗口中。

### 3. 行动后更新
完成任何阶段后，立即更新计划文件：
- 用 [x] 标记已完成的阶段
- 更新状态部分
- 记录遇到的任何错误

### 4. 存储，不要塞满
大型输出写入文件，而不是上下文。在工作记忆中只保留路径。

### 5. 记录所有错误
每个错误都记录在"遇到的错误"部分。这为未来任务积累知识。

## 何时使用此模式

**对以下情况使用三文件模式：**
- 多步骤任务（3+ 步骤）
- 研究任务
- 构建/创建某些东西
- 跨越多个工具调用的任务
- 任何需要组织的任务

**跳过以下情况：**
- 简单问题
- 单文件编辑
- 快速查找

## 要避免的反模式

| 不要 | 改为 |
|------|------|
| 使用 TodoWrite 进行持久化 | 创建 `task_plan.md` 文件 |
| 陈述目标一次就忘记 | 每次决策前重新阅读计划 |
| 隐藏错误并重试 | 将错误记录到计划文件 |
| 将所有内容塞入上下文 | 将大型内容存储在文件中 |
| 立即开始执行 | 首先创建计划文件 |
