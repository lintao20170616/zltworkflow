# Change: 添加 i18n 扫描工具

## Why
当前前端代码中存在大量硬编码的中文文本，缺乏国际化支持。为了支持多语言，需要提供工具自动扫描代码中的文本，提取需要翻译的内容，并生成标准的多语言本地化文件（local files）。这样可以：
- 自动化提取需要翻译的文本，减少手动工作
- 生成标准格式的多语言文件，便于与翻译平台集成
- 支持增量扫描，只提取新增或修改的文本
- 提供与现有翻译平台的集成能力

## What Changes
- **新增 i18n 扫描工具**
  - 扫描 Vue 组件文件（.vue）中的模板文本
  - 扫描 TypeScript/JavaScript 文件中的字符串字面量
  - 支持配置扫描规则（忽略某些文件、提取规则等）
  - 提取文本并生成翻译键（key）
  - 生成多语言本地化文件（JSON 格式）

- **新增扫描结果管理功能**
  - 扫描结果预览和确认
  - 支持手动编辑翻译键
  - 支持合并到现有本地化文件
  - 支持导出扫描结果到翻译平台项目

- **新增本地化文件管理**
  - 本地化文件存储结构（按语言代码组织）
  - 支持 JSON 格式的本地化文件
  - 提供 API 接口管理本地化文件
  - 支持从翻译平台同步翻译内容到本地化文件

## Impact
- **受影响的规范**：
  - 新增 `i18n-scanner` 功能规范
  - 扩展 `translation-platform` 功能规范（支持导入扫描结果）

- **受影响的代码**：
  - 后端：新增 `app/service/i18nScanner.js`、`app/controller/i18nScanner.js`
  - 前端：新增 `client/spa/apps/default/service/i18nScanner.ts`
  - 工具脚本：新增 `scripts/i18n-scanner.js`（CLI 工具）
  - 配置文件：新增 `config/i18n-scanner.js`（扫描配置）
  - 本地化文件：新增 `client/spa/apps/default/locales/` 目录结构
  - 路由：新增 `/api/i18n/scanner/*` 相关路由

- **数据模型扩展**：
  - 可能需要新增扫描记录表（可选，用于记录扫描历史）
  - 本地化文件存储在文件系统中，不存储在数据库
