## ADDED Requirements

### Requirement: 扫描 i18n $t 函数
系统 SHALL 提供代码扫描功能，自动扫描指定路径下的代码文件，提取 i18n 的 $t 函数中使用的翻译键。

#### Scenario: 扫描项目路径中的 $t 函数
- **WHEN** 用户指定项目路径
- **THEN** 系统扫描路径下的 .vue、.ts、.js 文件
- **AND** 提取所有 $t() 函数调用中的翻译键（如 `$t('key')`、`this.$t('key')`、`t('key')`）
- **AND** 返回扫描结果，包括文件路径、提取的翻译键、行号等信息
- **AND** 支持配置忽略规则（忽略某些文件或目录）

#### Scenario: 生成键值对 JSON
- **WHEN** 扫描完成后
- **THEN** 系统生成键值对 JSON 格式数据
- **AND** JSON 格式为：`{ "key1": "", "key2": "", ... }`
- **AND** 值为空字符串，等待后续在翻译系统中填写翻译内容

### Requirement: 导入扫描结果到翻译系统
系统 SHALL 支持将扫描生成的键值对 JSON 导入到翻译系统，自动创建翻译内容并生成翻译任务。

#### Scenario: 导入键值对 JSON 到翻译项目
- **WHEN** 用户提供扫描生成的键值对 JSON 和翻译项目 ID
- **THEN** 系统解析 JSON 数据
- **AND** 为每个键在指定项目中创建翻译内容记录
- **AND** 使用项目的源语言创建源文本记录（键作为源文本）
- **AND** 为项目的每个目标语言创建对应的翻译记录（初始状态为"待翻译"）
- **AND** 返回导入结果统计（成功数、失败数、错误信息）
- **AND** 如果翻译键已存在，则跳过或更新（根据配置）
