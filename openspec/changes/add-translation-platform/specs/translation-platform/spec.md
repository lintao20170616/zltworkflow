## ADDED Requirements

### Requirement: 翻译项目管理
系统应提供翻译项目的创建、编辑、删除和查询功能。每个项目应关联源语言和目标语言，支持多目标语言配置。

#### Scenario: 创建翻译项目
- **WHEN** 用户提供项目名称、描述、源语言和目标语言列表
- **THEN** 系统创建新项目并返回项目信息
- **AND** 项目状态默认为"进行中"

#### Scenario: 查询项目列表
- **WHEN** 用户请求项目列表
- **THEN** 系统返回所有项目，包括项目名称、状态、语言配置、创建时间等信息
- **AND** 支持按状态、创建时间、项目名称等条件筛选

#### Scenario: 更新项目状态
- **WHEN** 用户更新项目状态（进行中、已完成、已归档）
- **THEN** 系统更新项目状态并记录更新时间

### Requirement: 翻译内容管理
系统应提供翻译键值对的管理功能，支持多语言翻译内容的存储、查询和更新。每个翻译内容应关联项目、语言和翻译键。

#### Scenario: 创建翻译内容
- **WHEN** 用户提供项目ID、翻译键、源文本和目标语言的翻译文本
- **THEN** 系统创建翻译记录
- **AND** 确保同一项目下相同键和语言的组合唯一

#### Scenario: 查询翻译内容
- **WHEN** 用户按项目、语言、键等条件查询
- **THEN** 系统返回匹配的翻译内容列表
- **AND** 支持模糊搜索翻译键和翻译文本

#### Scenario: 批量导入翻译内容
- **WHEN** 用户上传包含翻译内容的文件（JSON、CSV等格式）
- **THEN** 系统解析文件并批量创建翻译记录
- **AND** 返回导入结果统计（成功数、失败数、错误信息）

#### Scenario: 批量导出翻译内容
- **WHEN** 用户选择项目和语言进行导出
- **THEN** 系统生成包含翻译内容的文件
- **AND** 支持多种格式（JSON、CSV、XLSX等）

### Requirement: 翻译任务管理
系统应提供翻译任务的创建、分配、跟踪和统计功能。任务应关联项目、翻译人员和审核人员。

#### Scenario: 创建翻译任务
- **WHEN** 用户指定项目、翻译人员、审核人员和截止日期
- **THEN** 系统创建任务并设置状态为"待翻译"
- **AND** 任务关联项目中的所有待翻译内容

#### Scenario: 更新任务状态
- **WHEN** 翻译人员或审核人员更新任务状态
- **THEN** 系统更新任务状态（待翻译、翻译中、待审核、已完成）
- **AND** 记录状态变更时间和操作人员

#### Scenario: 查看任务进度
- **WHEN** 用户查看任务详情
- **THEN** 系统显示任务进度统计（总数量、已完成数、待审核数等）
- **AND** 显示任务关联的翻译内容列表
