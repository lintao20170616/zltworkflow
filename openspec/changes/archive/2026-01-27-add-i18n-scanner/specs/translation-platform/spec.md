## MODIFIED Requirements

### Requirement: 翻译内容管理
系统 SHALL 提供翻译键值对的管理功能，支持多语言翻译内容的存储、查询和更新。每个翻译内容 SHALL 关联项目、语言和翻译键。系统 SHALL 支持从 i18n 扫描工具导入扫描结果。

#### Scenario: 创建翻译内容
- **WHEN** 用户在项目详情页的某个语言 Tab 下创建翻译内容
- **THEN** 系统自动关联当前项目和当前 Tab 对应的语言
- **AND** 用户提供翻译键、源文本和翻译文本
- **AND** 系统创建翻译记录并确保同一项目下相同键和语言的组合唯一

#### Scenario: 查询翻译内容
- **WHEN** 用户在项目详情页的某个语言 Tab 下查看翻译列表
- **THEN** 系统返回该项目下该语言的所有翻译内容
- **AND** 支持在该 Tab 内按翻译键和翻译文本进行模糊搜索
- **AND** 切换不同语言 Tab 时，显示对应语言的翻译列表

#### Scenario: 批量导入翻译内容
- **WHEN** 用户上传包含翻译内容的文件（JSON、CSV等格式）
- **THEN** 系统解析文件并批量创建翻译记录
- **AND** 返回导入结果统计（成功数、失败数、错误信息）

#### Scenario: 从 i18n 扫描工具导入键值对 JSON
- **WHEN** 用户提供扫描生成的键值对 JSON（格式：`{ "key1": "", "key2": "", ... }`）和翻译项目 ID
- **THEN** 系统解析 JSON 数据
- **AND** 系统获取项目的源语言和目标语言配置
- **AND** 为每个键在源语言下创建翻译记录（key 作为 sourceText，translatedText 为空）
- **AND** 为每个键在每个目标语言下创建翻译记录（状态为"待翻译"）
- **AND** 返回导入结果统计（成功数、失败数、错误信息）
- **AND** 如果翻译键已存在，则跳过（不重复创建）

#### Scenario: 批量导出翻译内容
- **WHEN** 用户选择项目和语言进行导出
- **THEN** 系统生成包含翻译内容的文件
- **AND** 支持多种格式（JSON、CSV、XLSX等）
