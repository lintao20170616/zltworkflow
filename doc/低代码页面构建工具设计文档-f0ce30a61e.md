# 低代码页面构建工具设计文档

## 1. 项目概述

### 1.1 项目背景
- **背景说明**：当前团队开发配置列表查询页面和表单页面时，存在大量重复性代码编写工作，开发效率低下。需要通过可视化拖拽配置的方式，降低开发门槛，提升开发效率。
- **问题分析**：传统页面开发方式需要编写大量重复的HTML/CSS/JS代码，配置逻辑相似度高但手工编写耗时，且难以统一维护。
- **解决方案概述**：构建一个基于JSON驱动的低代码页面构建工具，通过可视化拖拽方式配置页面布局和组件属性，生成标准JSON配置，再通过渲染组件动态生成最终页面。

### 1.2 项目目标
- **核心目标**：
  - 实现列表查询页面和表单页面的可视化配置
  - 支持PC端和移动端双端预览
  - 输出标准JSON配置供渲染组件使用
- **量化指标**：
  - 配置效率提升60%以上
  - 支持不少于20种常用组件
  - JSON配置生成时间<500ms

### 1.3 项目范围
- **包含的功能**：
  - 组件库拖拽配置
  - 画布布局编辑
  - 组件属性设置
  - PC/移动端实时预览
  - JSON配置生成与导出
  - 动态数据绑定的页面渲染组件
- **不包含的功能**：
  - 用户权限管理（第一版）
  - 版本管理与历史回滚（第一版）
  - 云端存储与多端同步（第一版）
  - 自定义组件扩展（第一版）

### 1.4 用户画像
- **主要用户群体**：前端开发工程师、全栈开发工程师
- **用户特征**：熟悉Vue3技术栈，需要快速开发配置类页面
- **使用场景**：开发后台管理系统的列表查询页、表单提交页、详情展示页

## 3. 系统架构设计

### 3.1 技术栈选型

#### 3.1.1 前端技术栈
- **框架**：Vue 3 + Composition API
- **UI库**：Element UI（作为预制组件库）
- **拖拽库**：Vue.Draggable / Sortable.js
- **状态管理**：Pinia（轻量级状态管理）
- **其他工具**：
  - TypeScript（类型安全）
  - Vite（构建工具）
  - JSON Schema（配置校验）
- **选型理由**：
  - Vue 3性能优异，组合式API更适合复杂状态管理
  - Element UI组件丰富，适合快速搭建管理类页面
  - Pinia相比Vuex更轻量，适合单页面状态管理

#### 3.1.2 后端技术栈
- **开发语言**：Node.js
- **Web框架**：Egg.js
- **ORM框架**：TypeORM / Sequelize（第一版暂无需数据库）
- **其他工具**：
  - TypeScript
  - Koa中间件（如需要）
- **选型理由**：
  - Egg.js提供企业级开发规范和插件生态
  - 团队熟悉Node.js生态
  - 第一版主要处理本地配置，后端可选

#### 3.1.3 数据库
- **数据库类型**：第一版无需数据库（本地存储）
- **具体产品**：LocalStorage / IndexedDB
- **选型理由**：
  - 第一版只支持本地使用，无需服务端持久化
  - 使用浏览器本地存储足够

#### 3.1.4 中间件（如需要）
- 第一版无需消息队列、缓存等中间件

## 4. 功能模块设计

### 4.1 模块划分

#### 4.1.1 模块列表
| 模块编号 | 模块名称 | 职责描述 |
|---------|---------|---------|
| M001 | 组件库模块 | 提供可拖拽的组件列表 |
| M002 | 画布编辑模块 | 负责组件的布局、拖放、编辑 |
| M003 | 属性配置模块 | 提供组件属性编辑界面 |
| M004 | 预览模块 | PC/移动端实时预览 |
| M005 | 配置生成模块 | 生成JSON配置 |
| M006 | 渲染器模块 | 接收JSON配置动态渲染页面 |

### 4.2 核心模块详细设计

#### 4.2.1 组件库模块
- **模块职责**：提供Element UI预制组件，支持拖拽到画布
- **核心功能**：
  - 组件分类展示（基础组件、表单组件、数据展示组件、布局组件）
  - 组件拖拽事件处理
- **输入输出**：
  - 输入：无（组件预设）
  - 输出：拖拽事件（组件类型、配置）
- **处理流程**：
  ```
  [用户拖拽] → [识别组件类型] → [发送拖拽数据] → [画布接收]
  ```
- **组件清单**（第一版）：
  - 基础组件：Button、Text、Icon
  - 表单组件：Input、Select、DatePicker、Checkbox、Radio、Switch
  - 数据展示：Table、Card、List、Tag
  - 布局组件：Row、Col、Container

#### 4.2.2 画布编辑模块
- **模块职责**：提供可视化编辑区域，支持组件的拖放、选中、移动、删除
- **核心功能**：
  - 组件拖放接收与放置
  - 组件选中与高亮
  - 组件移动排序
  - 组件删除
  - 层级调整
- **输入输出**：
  - 输入：拖拽事件、用户操作（点击/拖动/删除）
  - 输出：画布状态更新
- **处理流程**：
  ```
  [接收组件] → [生成组件实例] → [添加到画布] → [更新状态]
  [选中组件] → [显示属性面板] → [等待配置修改]
  [删除组件] → [移除实例] → [更新状态]
  ```
- **异常处理**：拖拽位置无效时提示用户

#### 4.2.3 属性配置模块
- **模块职责**：根据选中组件类型，动态展示可配置属性
- **核心功能**：
  - 根据组件类型加载属性配置表单
  - 实时同步属性修改到画布
  - 属性值校验
- **输入输出**：
  - 输入：选中组件信息、用户修改的属性值
  - 输出：更新后的组件配置
- **处理流程**：
  ```
  [组件选中] → [加载属性Schema] → [渲染属性表单]
  [用户修改] → [校验属性值] → [更新组件配置] → [画布重绘]
  ```
- **属性配置示例**：
  - Input组件：placeholder、disabled、type、maxlength
  - Button组件：type、size、icon、disabled

#### 4.2.4 预览模块
- **模块职责**：实时预览配置效果，支持PC/移动端切换
- **核心功能**：
  - PC端预览
  - 移动端预览（响应式布局）
  - 实时同步
- **输入输出**：
  - 输入：当前画布配置
  - 输出：预览视图
- **处理流程**：
  ```
  [配置更新] → [监听状态变化] → [重绘预览]
  [切换端] → [切换容器样式] → [重新渲染]
  ```

#### 4.2.5 配置生成模块
- **模块职责**：将画布配置序列化为标准JSON格式
- **核心功能**：
  - 递归遍历组件树
  - 生成标准JSON Schema
  - 导出JSON文件
- **输入输出**：
  - 输入：画布组件树状态
  - 输出：JSON配置字符串
- **处理流程**：
  ```
  [遍历组件树] → [提取组件配置] → [生成JSON] → [校验格式] → [输出]
  ```
- **JSON Schema示例**：
```json
{
  "version": "1.0.0",
  "page": {
    "title": "列表查询页",
    "layout": "container"
  },
  "components": [
    {
      "id": "comp_001",
      "type": "el-form",
      "props": {
        "labelWidth": "100px"
      },
      "children": [
        {
          "id": "comp_002",
          "type": "el-input",
          "props": {
            "placeholder": "请输入关键词",
            "v-model": "searchKeyword"
          }
        }
      ]
    }
  ]
}
```

#### 4.2.6 渲染器模块
- **模块职责**：接收JSON配置，动态渲染页面，支持数据绑定
- **核心功能**：
  - 解析JSON配置
  - 动态组件渲染
  - 数据绑定与事件处理
- **输入输出**：
  - 输入：JSON配置、动态数据对象
  - 输出：渲染后的Vue组件树
- **处理流程**：
  ```
  [接收JSON] → [解析配置] → [映射组件类型] → [动态渲染]
  [数据注入] → [双向绑定] → [事件绑定]
  ```
- **使用示例**：
```vue
<template>
  <DynamicRenderer
    :config="pageConfig"
    :data="formData"
    @submit="handleSubmit"
  />
</template>

<script>
import DynamicRenderer from './components/DynamicRenderer.vue'

export default {
  components: { DynamicRenderer },
  data() {
    return {
      pageConfig: { /* JSON配置 */ },
      formData: { /* 动态数据 */ }
    }
  }
}
</script>
```
