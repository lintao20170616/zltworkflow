## 🏗️ 架构设计规范

### 项目结构
```
client/spa/apps/default/
├── views/              # 页面视图
│   ├── auth/           # 认证模块
│   ├── dashboard/      # 仪表盘
│   ├── user/           # 用户模块
│   └── ...
├── components/         # 通用组件
│   ├── lowcode/        # 低代码组件
│   └── common/         # 通用基础组件
├── service/            # API服务封装
├── store/              # Pinia状态管理
├── router/             # 路由配置
├── i18n/               # 国际化配置
├── styles/             # 样式文件
├── types/              # TypeScript类型定义
└── utils/              # 工具函数
```

### 分层架构
```
┌─────────────────────────────────────────────────────┐
│ Views层: 页面视图，组合组件                          │
├─────────────────────────────────────────────────────┤
│ Components层: 可复用组件，UI展示                     │
├─────────────────────────────────────────────────────┤
│ Service层: API调用封装                              │
├─────────────────────────────────────────────────────┤
│ Store层: Pinia状态管理                              │
├─────────────────────────────────────────────────────┤
│ Router层: 路由配置                                  │
└─────────────────────────────────────────────────────┘
```

### 数据流规范
1. 用户操作 → 组件事件 → 调用Service → HTTP请求
2. 响应返回 → Service处理 → 更新Store → 组件重新渲染

## 📝 命名规范

### 文件命名
- **页面视图**: `xxx.vue` (如 `login.vue`, `index.vue`)
- **组件**: `Xxx.vue` (如 `Layout.vue`, `NavBarMenu.vue`)
- **Service**: `xxx.ts` (如 `auth.ts`, `chatbot.ts`)
- **Store**: `xxx.ts` (如 `user.ts`, `menu.ts`)
- **类型定义**: `xxx.d.ts` 或 `types/xxx.ts`

### 组件命名
- **大驼峰**: `Layout`, `NavBarMenu`, `UserList`
- **语义化**: 使用描述性名称，避免简写
- **组合命名**: 组件名由多个单词组成，描述其功能

### 变量命名
- **小驼峰**: `userName`, `userId`, `resultData`
- **常量**: `MAX_RETRY_COUNT`, `DEFAULT_PAGE_SIZE`
- **布尔值**: 使用`is`, `has`, `can`前缀（如 `isLoggedIn`, `hasPermission`）

### 函数命名
- **小驼峰**: `getUserInfo()`, `handleLogin()`
- **动词开头**: `get`, `set`, `update`, `delete`, `create`
- **事件处理**: 使用`handle`前缀（如 `handleSubmit`, `handleClick`）

## 📦 Vue组件开发规范

### 基本结构
```vue
<template>
  <div class="component-name">
    <!-- 组件模板 -->
    <el-button type="primary" @click="handleClick">
      {{ buttonText }}
    </el-button>
  </div>
</template>

<script setup lang="ts">
import { ref, computed, onMounted } from 'vue';
import { ElButton } from 'element-plus';

// Props定义
interface Props {
  buttonText?: string;
  disabled?: boolean;
}

const props = withDefaults(defineProps<Props>(), {
  buttonText: '按钮',
  disabled: false
});

// Emits定义
const emit = defineEmits<{
  (e: 'click', value: string): void;
}>();

// 响应式数据
const count = ref(0);

// 计算属性
const doubleCount = computed(() => count.value * 2);

// 方法
const handleClick = () => {
  count.value++;
  emit('click', 'clicked');
};

// 生命周期
onMounted(() => {
  console.log('Component mounted');
});
</script>

<style scoped>
.component-name {
  padding: 16px;
}
</style>
```

### Template规范
1. **根元素**: 每个组件必须有且只有一个根元素
2. **缩进**: 使用2个空格缩进
3. **属性顺序**: `id` → `class` → `props` → `events` → `directives`
4. **事件绑定**: 使用`@event`语法，方法名使用`handle`前缀
5. **条件渲染**: 使用`v-if`/`v-else`，避免在同一元素上使用`v-if`和`v-for`
6. **列表渲染**: 使用`v-for`必须添加`:key`

### Script规范
1. **Composition API**: 使用`<script setup>`语法
2. **导入顺序**: 第三方库 → 内部模块 → 样式文件
3. **Props**: 使用`defineProps`定义，`withDefaults`设置默认值
4. **Emits**: 使用`defineEmits`定义事件
5. **响应式**: 使用`ref`, `reactive`, `computed`
6. **生命周期**: 使用`onMounted`, `onUnmounted`等组合式API

### Style规范
1. **scoped**: 使用`<style scoped>`避免样式污染
2. **命名**: 使用BEM命名规范（`block__element--modifier`）
3. **缩进**: 使用2个空格缩进
4. **属性顺序**: 布局属性 → 盒模型 → 背景 → 文本 → 其他
5. **单位**: 使用`px`作为基础单位，`rem`作为响应式单位

## 📦 Service层开发规范

### 基本结构
```typescript
import http from '../utils/http';

interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
}

interface User {
  id: number;
  username: string;
  email: string;
  status: number;
}

export const authApi = {
  // 用户登录
  login: async (data: { username: string; password: string }) => {
    return await http.api.post<ApiResponse<User>>('/user/login', data);
  },
  
  // 获取当前用户信息
  getCurrentUser: async () => {
    return await http.api.get<ApiResponse<User>>('/user/info');
  },
  
  // 用户登出
  logout: async () => {
    return await http.api.post<ApiResponse<void>>('/user/logout');
  }
};
```

### API封装规范
1. **统一导出**: 每个模块导出一个API对象
2. **类型定义**: 为请求参数和响应数据定义TypeScript接口
3. **错误处理**: 使用统一的错误处理器（在`utils/http.ts`中定义）
4. **方法命名**: 使用语义化名称（`login`, `getList`, `create`, `update`, `delete`）

### HTTP请求规范
1. **请求前缀**: `/api`（在`utils/http.ts`中配置）
2. **超时时间**: 默认600秒（可根据需求调整）
3. **Credentials**: `include`（携带Cookie）
4. **Content-Type**: `application/json`

## 📦 Store层开发规范

### 基本结构
```typescript
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';

interface UserInfo {
  id: number;
  username: string;
  email?: string;
  status: number;
}

export const useUserStore = defineStore('user', {
  // 状态定义
  state: () => ({
    userInfo: null as UserInfo | null,
    isLoggedIn: false
  }),
  
  // 计算属性
  getters: {
    user: (state) => state.userInfo,
    userId: (state) => state.userInfo?.id
  },
  
  // 动作
  actions: {
    setUserInfo(userInfo: UserInfo) {
      this.userInfo = userInfo;
      this.isLoggedIn = true;
    },
    
    clearUserInfo() {
      this.userInfo = null;
      this.isLoggedIn = false;
    },
    
    async login(userInfo: UserInfo) {
      this.setUserInfo(userInfo);
    },
    
    async logout() {
      await authApi.logout();
      this.clearUserInfo();
    },
    
    async checkLoginStatus() {
      try {
        const res = await authApi.getCurrentUser();
        if (res && res.userInfo) {
          this.setUserInfo(res.userInfo);
          return true;
        }
        this.clearUserInfo();
        return false;
      } catch (error) {
        this.clearUserInfo();
        return false;
      }
    }
  }
});
```

### Store规范
1. **命名**: 使用`useXxxStore`命名
2. **状态**: 使用`state`函数定义初始状态
3. **计算属性**: 使用`getters`定义派生状态
4. **动作**: 使用`actions`定义状态变更方法
5. **异步操作**: 在`actions`中处理异步逻辑
6. **类型安全**: 使用TypeScript接口定义状态类型

## 🔗 路由开发规范

### 路由配置
```typescript
import { createRouter, createWebHistory, RouteRecordRaw } from 'vue-router';
import Layout from '@app/components/Layout.vue';

const routes: RouteRecordRaw[] = [
  {
    path: '/',
    name: 'Home',
    component: Layout,
    redirect: '/dashboard',
    children: [
      {
        path: '/dashboard',
        name: 'Dashboard',
        component: () => import('@app/views/dashboard/index.vue'),
        meta: { title: '仪表盘', icon: 'Dashboard' }
      },
      {
        path: '/userinfo',
        name: 'UserInfo',
        component: () => import('@app/views/user/info.vue'),
        meta: { title: '用户信息', icon: 'User' }
      }
    ]
  },
  {
    path: '/login',
    name: 'Login',
    component: () => import('@app/views/auth/login.vue'),
    meta: { title: '登录' }
  }
];

const router = createRouter({
  history: createWebHistory(),
  routes
});

export default router;
```

### 路由规范
1. **懒加载**: 使用动态导入`() => import()`实现路由懒加载
2. **元信息**: 使用`meta`字段存储页面标题、图标等信息
3. **布局**: 使用嵌套路由实现统一布局
4. **命名**: 使用`PascalCase`命名路由名称

### 路由守卫
```typescript
import router from './index';
import { useUserStore } from '@app/store';

router.beforeEach(async (to, from, next) => {
  const userStore = useUserStore();
  
  // 登录页面无需认证
  if (to.path === '/login') {
    next();
    return;
  }
  
  // 检查登录状态
  const isLoggedIn = await userStore.checkLoginStatus();
  
  if (isLoggedIn) {
    next();
  } else {
    next('/login');
  }
});
```

## 🎨 UI组件规范

### Element Plus使用规范
1. **组件导入**: 使用按需导入（通过`@zlt/zlt-admin-client`或手动导入）
2. **属性命名**: 使用`kebab-case`（如`el-button`, `el-input`）
3. **事件处理**: 使用`@event`语法
4. **表单验证**: 使用Element Plus内置验证规则

### 表单开发规范
```vue
<template>
  <el-form :model="form" :rules="rules" ref="formRef">
    <el-form-item label="用户名" prop="username">
      <el-input v-model="form.username" placeholder="请输入用户名" />
    </el-form-item>
    <el-form-item label="密码" prop="password">
      <el-input v-model="form.password" type="password" placeholder="请输入密码" />
    </el-form-item>
    <el-form-item>
      <el-button type="primary" @click="handleSubmit">提交</el-button>
    </el-form-item>
  </el-form>
</template>

<script setup lang="ts">
import { ref, reactive } from 'vue';
import { ElForm, ElFormItem, ElInput, ElButton } from 'element-plus';
import type { FormInstance, FormRules } from 'element-plus';

const formRef = ref<FormInstance>();

const form = reactive({
  username: '',
  password: ''
});

const rules: FormRules = {
  username: [
    { required: true, message: '请输入用户名', trigger: 'blur' },
    { min: 3, max: 50, message: '用户名长度为3-50个字符', trigger: 'blur' }
  ],
  password: [
    { required: true, message: '请输入密码', trigger: 'blur' },
    { min: 6, message: '密码至少6位', trigger: 'blur' }
  ]
};

const handleSubmit = async () => {
  if (!formRef.value) return;
  
  const isValid = await formRef.value.validate();
  if (!isValid) return;
  
  // 提交表单
};
</script>
```

## 📋 代码组织规范

### 文件组织
1. **按功能模块**: 将相关文件组织在同一目录下
2. **组件层级**: 通用组件放在`components/common/`，业务组件放在`views/xxx/components/`
3. **类型定义**: 统一放在`types/`目录下

### 导入路径
1. **别名配置**: 使用`@app`别名（在`vite.config.ts`中配置）
2. **相对路径**: 使用相对路径导入同目录或子目录文件
3. **绝对路径**: 使用别名导入跨目录文件

### 代码风格
1. **缩进**: 2个空格
2. **行长度**: 不超过120字符
3. **空行**: 适当使用空行分隔逻辑块
4. **注释**: 使用中文注释，复杂逻辑添加行内注释

## 📊 代码审查要点

### 组件审查要点
- [ ] Props定义是否完整，是否有默认值

### Service审查要点
- [ ] API封装是否完整
- [ ] 类型定义是否准确
- [ ] 是否有错误处理

### Store审查要点
- [ ] 状态定义是否合理
- [ ] 计算属性是否必要
- [ ] 异步操作是否正确处理

### 安全性审查要点
- [ ] 是否有XSS风险
- [ ] 是否有CSRF防护
