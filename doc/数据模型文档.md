## 📊 用户管理模块

### users（用户表）

#### 表结构
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INTEGER | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| password | VARCHAR(255) | NOT NULL | 密码（bcrypt加密） |
| email | VARCHAR(100) | NULL | 邮箱 |
| status | TINYINT(1) | NOT NULL, DEFAULT 1 | 状态：1-启用，0-禁用 |
| createdAt | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updatedAt | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 更新时间 |

#### 索引
| 索引名 | 字段 | 类型 |
|--------|------|------|
| PRIMARY | id | 主键 |
| username | username | 唯一 |
| email | email | 普通 |
| status | status | 普通 |

#### 模型定义
```javascript
// app/model/user.js
const User = app.model.define('User', {
  id: { type: INTEGER, primaryKey: true, autoIncrement: true },
  username: { type: STRING(50), allowNull: false, unique: true },
  password: { type: STRING(255), allowNull: false },
  email: { type: STRING(100), allowNull: true },
  status: { type: TINYINT, allowNull: false, defaultValue: 1 },
  createdAt: { type: DATE, allowNull: false, field: 'created_at' },
  updatedAt: { type: DATE, allowNull: false, field: 'updated_at' }
}, {
  tableName: 'users',
  timestamps: true,
  underscored: false
});
```

#### 关联关系
- **多对多**: User ↔ Role（通过user_roles关联表）

#### 生命周期钩子
```javascript
// 创建前自动加密密码
User.beforeCreate(async (user) => {
  if (user.password) {
    user.password = await bcrypt.hash(user.password, 10);
  }
});

// 更新前如果密码变更，重新加密
User.beforeUpdate(async (user) => {
  if (user.changed('password') && user.password) {
    user.password = await bcrypt.hash(user.password, 10);
  }
});
```

## 🔗 关联关系图

### 整体关联关系
```
users ───(多对多)─── roles
    │
    │ (一对多)
    ▼
conversations ───(一对多)─── messages
    │
    │ (一对多)
    ▼
translation_projects ───(一对多)─── translations
    │                              │
    │ (一对多)                     │ (外键)
    ▼                              ▼
translation_tasks ───────────── languages

systems ───(一对多)─── menus
    │
    │ (外键)
    ▼
languages
```

### 详细关联关系

#### 用户模块
```
User.belongsToMany(Role, {
  through: UserRole,
  foreignKey: 'userId',
  otherKey: 'roleId',
  as: 'roles'
});

Role.belongsToMany(User, {
  through: UserRole,
  foreignKey: 'roleId',
  otherKey: 'userId',
  as: 'users'
});
```

#### 聊天机器人模块
```
Conversation.hasMany(Message, {
  foreignKey: 'conversationId',
  as: 'messages'
});

Message.belongsTo(Conversation, {
  foreignKey: 'conversationId',
  as: 'conversation'
});

Conversation.belongsTo(User, {
  foreignKey: 'userId',
  as: 'user'
});
```

#### 翻译平台模块
```
TranslationProject.hasMany(Translation, {
  foreignKey: 'projectId',
  as: 'translations'
});

Translation.belongsTo(TranslationProject, {
  foreignKey: 'projectId',
  as: 'project'
});

TranslationProject.hasMany(TranslationTask, {
  foreignKey: 'projectId',
  as: 'tasks'
});

TranslationTask.belongsTo(TranslationProject, {
  foreignKey: 'projectId',
  as: 'project'
});

Translation.belongsTo(Language, {
  foreignKey: 'languageId',
  as: 'language'
});

TranslationProject.belongsTo(Language, {
  foreignKey: 'sourceLanguageId',
  as: 'sourceLanguage'
});
```

## 📋 索引优化建议

### 现有索引
| 表名 | 索引字段 | 索引类型 | 用途 |
|------|----------|----------|------|
| users | username | UNIQUE | 登录查询 |
| users | email | INDEX | 用户搜索 |
| users | status | INDEX | 状态筛选 |
| conversations | userId | INDEX | 用户会话查询 |
| messages | conversationId | INDEX | 会话消息查询 |
| messages | createdAt | INDEX | 时间排序 |
| translation_projects | sourceLanguageId | INDEX | 语言筛选 |
| translations | projectId | INDEX | 项目翻译查询 |
| translations | projectId, key, languageId | UNIQUE | 唯一性约束 |

### 潜在优化建议
1. **translation_tasks**: 添加 `(projectId, status)` 复合索引，用于任务列表查询
2. **messages**: 添加 `(conversationId, createdAt)` 复合索引，优化消息排序查询
3. **translations**: 添加 `(projectId, status)` 复合索引，优化状态筛选

## 🔐 安全性考虑

### 数据加密
- **密码字段**: 使用bcryptjs加密存储
- **Session**: 使用Redis加密存储（encrypt: true）

### 输入验证
- **参数验证**: Controller层严格校验
- **SQL注入**: 使用Sequelize ORM，避免拼接SQL
- **XSS防护**: 前端和后端双重防护

### 访问控制
- **外键约束**: 确保数据完整性
- **权限验证**: OAuth中间件统一验证
