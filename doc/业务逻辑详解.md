## 🏗️ 系统架构

### 分层架构
```
┌─────────────────────────────────────────────────────────────┐
│                     前端层 (Vue 3)                          │
│  Views ──► Components ──► Service ──► Store ──► Router     │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTP/REST API
┌──────────────────────▼──────────────────────────────────────┐
│                    后端层 (Egg.js)                          │
│  Controller ──► Service ──► Model ──► MySQL/Redis          │
└─────────────────────────────────────────────────────────────┘
```

### 数据流
1. **前端请求**: 用户操作 → Vue组件 → Service层 → HTTP请求
2. **后端处理**: Router → Controller → Service → Model → 数据库
3. **响应返回**: 数据库 → Model → Service → Controller → HTTP响应 → 前端

## 🔐 用户管理模块

### 功能概述
- 用户注册、登录、登出
- 用户信息管理
- 用户列表查询
- 用户状态管理（启用/禁用）

### 核心流程

#### 用户登录流程
```
前端登录页 → POST /api/user/login
              ↓
UserController.login()
              ↓
UserService.validateUser(username, password)
              ↓
  ┌──────────────────────────────┐
  │ 1. 查询用户是否存在           │
  │ 2. 检查用户状态是否启用       │
  │ 3. 使用bcrypt验证密码         │
  │ 4. 返回验证结果              │
  └──────────────────────────────┘
              ↓
验证通过 → ctx.session.user = {...} → 返回用户信息
验证失败 → 返回错误码和提示信息
```

#### 用户注册流程
```
前端注册页 → POST /api/user/register
              ↓
UserController.register()
              ↓
参数验证（用户名、密码、邮箱格式）
              ↓
UserService.createUser(username, password, email)
              ↓
  ┌──────────────────────────────┐
  │ 1. 检查用户名是否已存在       │
  │ 2. 创建用户（密码自动加密）   │
  │ 3. 返回创建结果              │
  └──────────────────────────────┘
              ↓
创建成功 → 自动登录 → 返回用户信息
创建失败 → 返回错误码和提示信息
```

### 数据模型
```javascript
// User模型结构 (app/model/user.js)
{
  id: INTEGER (主键, 自增),
  username: STRING(50) (唯一),
  password: STRING(255) (bcrypt加密),
  email: STRING(100),
  status: TINYINT (1-启用, 0-禁用),
  createdAt: DATE,
  updatedAt: DATE
}
```

### 关联关系
- User ↔ Role (多对多，通过UserRole关联表)

### 核心方法

#### UserService.validateUser
**功能**: 验证用户名和密码  
**参数**: `username` (用户名), `password` (密码)  
**返回**: `{ success: boolean, message: string, user?: User }`

#### UserService.createUser
**功能**: 创建新用户  
**参数**: `username`, `password`, `email`  
**返回**: `{ success: boolean, message: string, user?: Object }`

## 🔑 权限控制模块

### 功能概述
- 基于角色的访问控制（RBAC）
- OAuth中间件统一认证
- 接口白名单机制
- 菜单权限绑定

### 认证流程
```
HTTP请求 → oauth中间件
            ↓
    检查是否在白名单
            ↓
    ┌─────┴─────┐
    │           │
  在白名单    不在白名单
    │           ↓
  直接放行   检查Session
                  ↓
            ┌────┴────┐
            │         │
        Session存在  Session不存在
            │         ↓
        继续处理    返回401错误
```

### OAuth中间件配置
```javascript
// config/config.default.js
config.oauth = {
  enable: true,
  whitelist: [
    '/',
    '/api/user/login',
    '/api/user/register',
    '/api/user/logout',
    // ... 其他公开接口
  ],
  userKey: 'user',
};
```

### 白名单机制
- **登录接口**: `/api/user/login`, `/api/user/register`
- **静态资源**: `/assets/`, `/healthCheck.html`
- **工具接口**: `/api/chatbot/test-ollama`, `/api/translation/push-default-json`

### 权限验证逻辑
```javascript
// app/lib/plugin/oauth/app/middleware/oauth.js
const isWhitelisted = whitelist.some((pattern) => {
  if (typeof pattern === 'string') {
    return path === pattern || path.startsWith(pattern);
  }
  if (pattern instanceof RegExp) {
    return pattern.test(path);
  }
  return false;
});
```

```javascript
// 递归构建菜单树
function buildMenuTree(menus, parentId = 0) {
  return menus
    .filter(menu => menu.parentId === parentId)
    .map(menu => ({
      ...menu,
      children: buildMenuTree(menus, menu.id)
    }));
}
```

## 🤖 聊天机器人模块

### 功能概述
- 基于Ollama的智能对话
- 会话管理（多会话支持）
- 消息历史记录
- 上下文记忆和摘要优化
- 自动降级机制

### 核心流程

#### 发送消息流程
```
用户发送消息 → POST /api/chatbot/send
                ↓
ChatbotService.sendMessage()
                ↓
  ┌──────────────────────────────┐
  │ 1. 创建用户消息记录           │
  │ 2. 调用Ollama生成回复         │
  │    ├─ 构建上下文记忆          │
  │    ├─ 调用AI模型             │
  │    └─ 处理响应               │
  │ 3. 创建机器人消息记录         │
  │ 4. 更新会话标题              │
  │ 5. 更新上下文缓存            │
  └──────────────────────────────┘
                ↓
返回用户消息和机器人回复
```

### 上下文记忆机制

#### 记忆摘要（Summary模式）
```
最近N条消息 → 摘要阈值触发 → 生成记忆摘要 → 缓存到Redis
                ↓
下次请求时 → 加载记忆摘要 + 最近消息 → 构建完整上下文
```

#### 记忆配置
```javascript
// OllamaService.getMemoryConfig()
{
  enable: true,
  type: 'summary',
  maxHistoryLength: 5,
  summaryThreshold: 1,
  summaryModel: 'deepseek-r1:7b',
}
```

### 降级机制
```javascript
// ChatbotService.sendMessage()
if (ctx.app.config.ollama?.enabled) {
  try {
    // 调用Ollama API
    botResponse = await ctx.service.ollama.ollamaChat(...);
  } catch (error) {
    // Ollama调用失败，使用规则回复
    botResponse = await this.generateBotResponse(conversationId, content);
  }
} else {
  // 未启用Ollama，使用规则回复
  botResponse = await this.generateBotResponse(conversationId, content);
}
```

### 规则回复示例
```javascript
// ChatbotService.getSimpleResponse()
if (message.includes('你好') || message.includes('hello')) {
  return '您好！我是聊天机器人，很高兴为您服务。';
}
if (message.includes('时间')) {
  return `当前时间是：${new Date().toLocaleString('zh-CN')}`;
}
// ... 更多规则
```

### 数据模型

#### Conversation（会话）
```javascript
{
  id: INTEGER,
  userId: INTEGER,
  title: STRING(200),
  summary: TEXT (对话摘要),
  createdAt: DATE,
  updatedAt: DATE,
}
```

#### Message（消息）
```javascript
{
  id: INTEGER,
  conversationId: INTEGER,
  role: STRING(20) (user/bot),
  content: TEXT,
  createdAt: DATE,
}
```

```javascript
{
  id: INTEGER,
```

```javascript
  completedCount: INTEGER,
  dueDate: DATE,
  createdAt: DATE,
  updatedAt: DATE,
```

### 翻译键值对管理
- 每个项目有唯一的翻译键（key）
- 同一个key可以有多个语言版本的翻译
- `project_id + key + language_id` 构成唯一约束

### i18n集成
- 扫描项目中的翻译文本：`npm run i18n:scan`
- 推送翻译到后端：`npm run i18n:push`
- 拉取翻译到前端：`npm run i18n:pull`

## 📝 统一响应格式

### 成功响应
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 失败响应
```json
{
  "code": 400,
  "message": "参数错误",
  "data": null
}
```

### 状态码说明
| 状态码 | HTTP状态 | 说明 |
|--------|----------|------|
| 0 | 200 | 成功 |
| 400 | 400 | 请求参数错误 |
| 401 | 401 | 未认证/登录过期 |
| 403 | 403 | 无权限 |
| 500 | 500 | 服务器错误 |
