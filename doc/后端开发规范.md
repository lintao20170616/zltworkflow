# ZLT Workflow - 后端开发规范

## 📖 文档概述

本文档定义了后端开发的规范和最佳实践，确保代码质量、一致性和可维护性。

**目标读者**: 后端开发人员、AI 助手、代码审查者  
**更新时间**: 2026-06-26

---

## 🏗️ 架构设计规范

### 分层架构

```
┌─────────────────────────────────────────────────────┐
│ Controller层: 处理HTTP请求，参数校验，调用Service   │
├─────────────────────────────────────────────────────┤
│ Service层: 业务逻辑处理，数据转换                   │
├─────────────────────────────────────────────────────┤
│ Model层: 数据模型定义，数据库交互                   │
└─────────────────────────────────────────────────────┘
```

### 职责划分

- **Controller**: 仅处理 HTTP 请求和响应，不包含业务逻辑
- **Service**: 封装业务逻辑，可被多个 Controller 复用
- **Model**: 定义数据模型和关联关系，封装数据库操作

### 数据流规范

1. Controller 接收请求 → 参数校验 → 调用 Service
2. Service 处理业务逻辑 → 调用 Model → 返回结果
3. Controller 组装响应 → 返回给前端

## 📝 命名规范

### 文件命名

- **Controller**: `xxx.js` (如 `user.js`, `chatbot.js`)
- **Service**: `xxx.js` (如 `user.js`, `chatbot.js`)
- **Model**: `xxx.js` (如 `user.js`, `conversation.js`)
- **配置文件**: `config.xxx.js`
- **迁移文件**: `序号_描述.sql` (如 `007_create_xxx_table.sql`)

### 类命名

- **Controller**: `XxxController` (如 `UserController`)
- **Service**: `XxxService` (如 `UserService`)
- **Model**: `Xxx` (如 `User`, `Conversation`)

### 方法命名

- **Controller 方法**: 使用 HTTP 动作或业务动作命名
  ```javascript
  async login()       // 登录
  async create()      // 创建
  async update()      // 更新
  async delete()      // 删除
  async list()        // 列表
  ```
- **Service 方法**: 使用动词开头，描述具体动作
  ```javascript
  async validateUser()      // 验证用户
  async createUser()        // 创建用户
  async getUserInfo()       // 获取用户信息
  async getUserList()       // 获取用户列表
  ```

### 变量命名

- **小驼峰**: `userName`, `userId`, `resultData`
- **常量**: `MAX_RETRY_COUNT`, `DEFAULT_PAGE_SIZE`
- **临时变量**: `temp`, `tmp`, `result`, `data`

## 📦 Controller 开发规范

### 基本结构

```javascript
const Controller = require('egg').Controller;

class XxxController extends Controller {
  async actionName() {
    const { ctx } = this;

    // 参数获取
    const { param1, param2 } = ctx.request.body;

    // 参数校验
    if (!param1 || !param2) {
      ctx.body = { code: 400, message: '参数错误', data: null };
      ctx.status = 400;
      return;
    }

    try {
      // 调用Service
      const result = await ctx.service.xxx.methodName(param1, param2);

      // 处理结果
      if (!result.success) {
        ctx.body = { code: 500, message: result.message, data: null };
        ctx.status = 500;
        return;
      }

      // 返回成功响应
      ctx.body = {
        code: 0,
        message: 'success',
        data: result.data,
      };
    } catch (error) {
      // 异常处理
      ctx.logger.error('[XxxController] actionName error:', error);
      ctx.body = { code: 500, message: '服务器错误', data: null };
      ctx.status = 500;
    }
  }
}

module.exports = XxxController;
```

### 参数校验规范

1. **必填参数**: 必须检查是否为空
2. **格式校验**: 邮箱、手机号等使用正则校验
3. **长度限制**: 字符串长度检查
4. **数值范围**: 数值类型的范围检查

### 响应格式规范

```javascript
// 成功响应
ctx.body = {
  code: 0,
  message: 'success',
  data: result.data,
};

// 失败响应
ctx.body = {
  code: 400,
  message: '参数错误',
  data: null,
};
ctx.status = 400;
```

### 错误处理规范

1. **参数错误**: 返回 400 状态码
2. **认证失败**: 返回 401 状态码（由 OAuth 中间件处理）
3. **业务错误**: 返回 500 状态码，包含具体错误信息
4. **异常捕获**: 使用 try-catch 包裹所有业务逻辑

### 日志记录规范

```javascript
// 错误日志
ctx.logger.error('[XxxController] actionName error:', error);

// 警告日志
ctx.logger.warn('[XxxController] actionName warning:', message);

// 信息日志
ctx.logger.info('[XxxController] actionName info:', data);
```

## 📦 Service 开发规范

### 基本结构

```javascript
const { Service } = require('egg');

class XxxService extends Service {
  async methodName(param1, param2) {
    const { ctx } = this;

    try {
      // 业务逻辑处理
      const data = await ctx.model.Xxx.findOne({
        where: { key: param1 },
      });

      // 返回成功结果
      return {
        success: true,
        data,
      };
    } catch (error) {
      // 记录错误日志
      ctx.logger.error('[XxxService] methodName error:', error);

      // 返回失败结果
      return {
        success: false,
        message: '操作失败',
      };
    }
  }
}

module.exports = XxxService;
```

### 返回值规范

```javascript
// 成功
return { success: true, data: result };

// 成功（无数据）
return { success: true };

// 失败
return { success: false, message: '错误描述' };

// 失败（详细信息）
return { success: false, message: '错误描述', code: 1001 };
```

### 事务处理规范

```javascript
async methodName() {
  const { ctx } = this;
  const transaction = await ctx.model.transaction();

  try {
    // 事务操作
    await ctx.model.Xxx.create(data1, { transaction });
    await ctx.model.Yyy.create(data2, { transaction });

    // 提交事务
    await transaction.commit();
    return { success: true };
  } catch (error) {
    // 回滚事务
    await transaction.rollback();
    ctx.logger.error('[XxxService] methodName error:', error);
    return { success: false, message: '操作失败' };
  }
}
```

### 业务逻辑规范

1. **单一职责**: 每个 Service 方法只做一件事
2. **可复用性**: 将通用逻辑提取为独立方法
3. **事务控制**: 涉及多表操作时使用事务
4. **异常处理**: 在 Service 层捕获并处理业务异常

## 📦 Model 开发规范

### 基本结构

```javascript
'use strict';

module.exports = (app) => {
  const { STRING, INTEGER, TINYINT, DATE, TEXT, JSON } = app.Sequelize;

  const Xxx = app.model.define(
    'Xxx',
    {
      id: {
        type: INTEGER,
        primaryKey: true,
        autoIncrement: true,
        comment: '主键ID',
      },
      name: {
        type: STRING(100),
        allowNull: false,
        comment: '名称',
      },
      status: {
        type: TINYINT,
        allowNull: false,
        defaultValue: 1,
        comment: '状态：1-启用，0-禁用',
      },
      createdAt: {
        type: DATE,
        allowNull: false,
        field: 'created_at',
        comment: '创建时间',
        defaultValue: app.Sequelize.literal('CURRENT_TIMESTAMP'),
      },
      updatedAt: {
        type: DATE,
        allowNull: false,
        field: 'updated_at',
        comment: '更新时间',
        defaultValue: app.Sequelize.literal('CURRENT_TIMESTAMP'),
      },
    },
    {
      tableName: 'xxx',
      timestamps: true,
      underscored: false,
    },
  );

  // 关联关系
  Xxx.associate = function () {
    if (app.model.Yyy) {
      Xxx.hasMany(app.model.Yyy, {
        foreignKey: 'xxxId',
        as: 'yyys',
      });
    }
  };

  return Xxx;
};
```

### 字段类型规范

| 类型      | 用途             | 示例                  |
| --------- | ---------------- | --------------------- |
| INTEGER   | 主键、外键、数值 | id, userId, status    |
| STRING(n) | 字符串，限制长度 | name(100), email(100) |
| TEXT      | 长文本           | content, description  |
| TINYINT   | 状态、布尔值     | status(1/0), enabled  |
| DATE      | 时间戳           | createdAt, updatedAt  |
| JSON      | JSON 格式数据    | config, meta          |

### 字段命名规范

- **驼峰命名**: `userName`, `createdAt`
- **数据库字段**: 自动转换为下划线形式（通过 Sequelize 配置）
- **注释**: 每个字段必须添加 comment 说明用途

### 关联关系规范

```javascript
// 一对一
Xxx.hasOne(Yyy, { foreignKey: 'xxxId', as: 'yyy' });
Yyy.belongsTo(Xxx, { foreignKey: 'xxxId', as: 'xxx' });

// 一对多
Xxx.hasMany(Yyy, { foreignKey: 'xxxId', as: 'yyys' });
Yyy.belongsTo(Xxx, { foreignKey: 'xxxId', as: 'xxx' });

// 多对多
Xxx.belongsToMany(Yyy, {
  through: Zzz,
  foreignKey: 'xxxId',
  otherKey: 'yyyId',
  as: 'yyys',
});
Yyy.belongsToMany(Xxx, {
  through: Zzz,
  foreignKey: 'yyyId',
  otherKey: 'xxxId',
  as: 'xxxs',
});
```

### 生命周期钩子

```javascript
// 创建前钩子
Xxx.beforeCreate(async (instance) => {
  // 处理逻辑（如密码加密）
});

// 更新前钩子
Xxx.beforeUpdate(async (instance) => {
  // 处理逻辑（如密码重新加密）
});

// 删除前钩子
Xxx.beforeDestroy(async (instance) => {
  // 处理逻辑（如级联删除）
});
```

## 🔗 路由开发规范

### 路由注册

```javascript
// app/router.js
module.exports = (app) => {
  const { router, controller } = app;

  // 用户管理
  router.post('/api/user/login', controller.user.login);
  router.post('/api/user/register', controller.user.register);
  router.get('/api/user/info', controller.user.getCurrentUser);
  router.post('/api/user/logout', controller.user.logout);
  router.get('/api/user/list', controller.user.getUserList);
};
```

### RESTful 规范

| 方法   | 路径                       | 功能     |
| ------ | -------------------------- | -------- |
| GET    | `/api/resource`            | 获取列表 |
| GET    | `/api/resource/:id`        | 获取详情 |
| POST   | `/api/resource`            | 创建资源 |
| PUT    | `/api/resource/:id`        | 更新资源 |
| DELETE | `/api/resource/:id`        | 删除资源 |
| PATCH  | `/api/resource/:id/status` | 更新状态 |

### 命名规范

- **资源名称**: 使用复数形式（`users`, `systems`, `menus`）
- **路径格式**: `/api/模块名/资源名/子资源`
- **ID 参数**: 使用`:id`表示资源标识

### 认证规则

- **默认**: 所有接口需要 OAuth 认证
- **白名单**: 在`config/config.default.js`中配置公开接口

## 🗄️ 数据库开发规范

### 数据库设计规范

1. **主键**: 使用自增 INTEGER 类型，命名为`id`
2. **外键**: 命名为`xxxId`，建立外键约束
3. **时间戳**: 每个表必须包含`created_at`和`updated_at`
4. **状态字段**: 使用 TINYINT 类型，1 表示启用，0 表示禁用
5. **字符集**: 使用 utf8mb4 支持 emoji
6. **索引**: 关键字段建立索引（外键、唯一字段、查询字段）

### 迁移脚本规范

```sql
-- database/migrations/008_create_xxx_table.sql
CREATE TABLE `xxx` (
  `id` int(11) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `name` varchar(100) NOT NULL COMMENT '名称',
  `description` text COMMENT '描述',
  `status` tinyint(1) NOT NULL DEFAULT '1' COMMENT '状态：1-启用，0-禁用',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='xxx表';
```

### 迁移执行

```bash
npm run migrate
```

## 🔐 安全规范

### 密码安全

1. **加密算法**: 使用 bcryptjs，10 轮加密
2. **存储**: 不存储明文密码
3. **验证**: 使用`bcrypt.compare()`验证

### Session 安全

1. **加密存储**: `encrypt: true`
2. **HTTPOnly**: 防止 XSS 攻击
3. **密钥管理**: 使用固定密钥用于父子系统共享

### CSRF 防护

1. **启用 CSRF**: 在配置中启用
2. **白名单**: 公开接口加入白名单
3. **Token 验证**: 前端请求携带 CSRF Token

### 输入验证

1. **参数校验**: Controller 层严格校验参数
2. **SQL 注入**: 使用 Sequelize ORM，避免拼接 SQL
3. **XSS 防护**: 对用户输入进行转义处理

## 🚀 性能规范

### 数据库优化

1. **索引优化**: 高频查询字段建立索引
2. **分页查询**: 使用`limit`和`offset`避免全表扫描
3. **批量操作**: 使用`bulkCreate`、`bulkUpdate`减少数据库交互
4. **连接池**: 合理配置数据库连接池大小

### 缓存优化

1. **Session 缓存**: 使用 Redis 存储 Session
2. **业务缓存**: 高频访问数据缓存到 Redis
3. **缓存策略**: 设置合理的过期时间（TTL）

### 代码优化

1. **异步操作**: 使用`async/await`避免阻塞
2. **批量查询**: 合并多次查询为一次
3. **避免 N+1 查询**: 使用`include`进行关联查询

## 🔧 配置规范

### 配置文件结构

```
config/
├── config.default.js    # 默认配置（生产环境）
├── config.local.js      # 本地配置（开发环境，不提交）
```

### 配置项规范

1. **环境变量**: 使用`process.env.xxx`读取环境变量
2. **默认值**: 配置项必须有默认值
3. **注释**: 关键配置项添加注释说明
4. **安全**: 敏感信息（密码、密钥）不硬编码，使用环境变量
